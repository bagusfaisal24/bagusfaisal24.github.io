Pengalaman 1 Tahun Bekerja Sebagai Software Engineer

November 22, 2019 2 minutes read
software engineer

Sebelum saya menuliskan pengalaman 1 tahun bekerja sebagai software engineer, sebelumnya mohon maaf apabila tulisan saya ini jelek, sudah lama tidak menulis artikel.

Oke mulai ya, saat kuliah saya mengambil jurusan Teknik Informatika di Kampus Universitas Ahmad dahlan di Yogyakarta. Saat masuk perkuliahan saya sedikit kaget karena perkuliahan semua membahas ke programming, karena saat SMA saya di IPA sedikit pengalaman di dunia TI.

Singkatnya saya lulus 4th tepat dengan gelar S.Kom dan mendapatkan pekerjaan pertama saya di Jakarta, diperusahan konsultan Dana pensiun. Perusahaan saya ini sedang mengembangkan Sistem dana pensiun yang dimana nanti pekerja pekerja swasta bisa mendaftar dan dapat menikmati hari tua nya dengan uang pensiun nya.

Sistem yang dibangun termasuk advanced dan monolitik menggunakan bahasa pemrograman Java, saya tidak terlalu kaget karena skripsi saya juga menggunakan bahasa Java mungkin ini jalan ninjaku bekerja sebagai java developer. Hal-hal dibawah ini sedikit pengalaman saya sebagai software engineer:

  1. Clean Code Principles

Kode bersih, perbedaan antara dunia perkuliahan dan professional kita ngoding tidak hanya yang penting jadi, tetapi code yang kita buat sebisa mungkin bisa dibaca dan dilanjutkan orang lain.

  1. 80% mining informations 20% implement.

Yang menjadi lama kita deliver tugas yang dikasih karena saat kita terima tugas ialah langsung dikerjakan bukan melalui proses mengumpulkan data sebanyak banyaknya terlebih dahulu. Jika sudah dapat banyak data dan informasi apa saja yang akan kita kerjakan barulah mulai implement / coding itu menjadi lebih effektif saat kita mengerjakan tugas yang didapat.

  1. Product Owner

Saat kita dapat dapat task pekerjaan bertanggung jawab lah apa yang menjadi tanggung jawab kita artinya harus mengerti dari a sampai z task yang akan dikerjakan. Satu lagi yang dapat saya bagikan jika kita bekerja dilingkungan 1 tim yang artinya banyak orang disana, sebisa mungkin kita mengerti fitur fitur yang dikerjakan orang lain. Jadi jika orang tersebut sudah tidak bekerja ditim kita masih bisa membackup, apalagi bekerja sebagai developer / software engineer 1 project itu bisa lebih dari 5 orang yang berkontribusi.
