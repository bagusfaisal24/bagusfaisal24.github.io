Tips Dasar Debugging Program

December 15, 2019 3 minutes read
dev • debugging
software engineer

Hay gaess, gimana kabarnya semoga sehat teruuss ya? keep learning :))

Kali ini saya akan berbagi pengalaman debugging program/ aplikasi, keseharian saya ngoding dengan bahasa pemrograman Java. IDEA yang saya pakai intelij ini bukan endorse ya hehe.

Unggul nya intelij sudah terdapat fitur debug di dalamnya, kalau temen - temen belum memakai IDEA yang sudah ada fitur debug, temen - temen bisa tambahkan aplikasi tambahan untuk debugging kalau di PHP ada namanya xdebug dan lainnya.

Berikut tips tips dasar saat debugging program:

Kenali masalahnya (Problem identify)

Hal pertama yang perlu diperhatikan saat mau debugging yakni kenali masalahnya terlebih dahulu. Maksudnya disini kalau teman teman mau debug fitur atau fungsi yang berjalan catat expected result atau output yang diharapkan dari program yang dibuat. Jangan asal langsung debug karena bukannya nanti solved problem malah nambah - nambah masalah XD :))

Bug type data or logic function ?

Pengalaman yang sering saya temukan mungkin temen temen sering menemukan juga bug seperti Null Pointer Exception bug ini sering temui karena biasanya programmer tidak menambahkan handler untuk type data yang nanti kemungkinan bakal null artinya ada yang harusnya di set / insert ke suatu model tapi data tersebut tidak ada maka Null Pointer Exception bakal menggangu anda. Type bug lain seperti miss logic function misalkan suatu function tambah yang didalamnya harusnya proses matematic tambah tapi malah proses pengurangan nah ini termasuk bug miss logic function. Beberapa waktu lalu saya dapatkan bug simple sih sebenarnya karena kurangnya komunikasi dengan bagian frontend data yang harusnya diterima sebagai id dibackend, tapi difrontend data payload yang dikirim code jadi dibackend bakal nangkap ini sebagai Null Pointer Exception karena searching datanya tidak ada yang sesuai.

Contoh Cara debugging

Contoh debugging dari case sederhana yang saya temuin beberapa waktu yang lalu. Data payload yang dikirimkan oleh frontend tidak match dengan yang backend butuhin.

  • Kenali masalahnya (Problem indentify)

Terdapat masalah saat menambahkan data, terdapat message error Null Pointer Exception, hasil yang diharapkan yaitu data bisa ditambahkan.

  • Bug type data or logic function

Dari identify masalah dapat kita ambil kesimpulan kemungkinan ini termasuk bug type data karena terdapat message error Null Pointer Exception

  • Mulai debugging

Proses yang mengalami bug yakni saat menambahkan data, jadi saya langsung jalanin debugger dari intelij dan bikin break point pada saat proses menambahkan data, nah break point ini cara untuk mengetahui pada baris kode mana yang mengalami masalah.

gambar diatas merupakan contoh potongan program yang sedang didebug. Bug tersebut ada di baris 80 data yang harus diterima yakni id tetapi data payload yang dikirimkan dari frontend berupa code. Jadi saat insert ke database mengalami error karena data yang dikirimkan tidak ada atau Null.

Mungkin diatas beberapa pengalaman saat debugging program yang bisa saya bagikan, apabila ada kekurangan dalam penulisan ngapunten nggih :D. KEEP LEARNING
